# Sprint Reports

Reportes de cada sprint de desarrollo.

## Estructura

Cada sprint report sigue el formato: `YYYY-MM-DD-sprint-N.md`

## Contenido de cada reporte

- Objetivo del sprint
- Items completados
- Items no completados (con razon)
- Velocity (story points o items)
- Retrospectiva (que funciono, que mejorar)
- Plan para siguiente sprint
